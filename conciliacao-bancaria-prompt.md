## Funcionalidade a Implementar: Conciliação Bancária

### Visão Geral

Permitir que o usuário importe um arquivo **OFX** (Open Financial Exchange) exportado do seu banco, e o sistema realize automaticamente a conciliação com as **Contas a Pagar/Receber** (`tb_conta`) já cadastradas, registrando as baixas em lote ao final.

---

## Estratégia de Vinculação OFX ↔ Contas a Pagar

> **Recomendação de Implementação (melhor abordagem):**
>
> Use **correspondência por score** com múltiplos critérios ponderados. A transação do OFX é vinculada à conta a pagar/receber que atingir maior pontuação:
>
> | Critério | Peso |
> |---|---|
> | Valor idêntico (`TRNAMT`) | 40 pts |
> | Valor com tolerância ±R$0,05 (centavos) | 20 pts |
> | Data OFX = `data_vencimento` ±3 dias | 30 pts |
> | Similaridade de texto `MEMO` vs `descricao` (≥ 60%) | 20 pts |
> | Parceiro com nome similar ao `MEMO` | 10 pts |
>
> **Threshold mínimo:** score ≥ 50 para sugerir vinculação automática. Abaixo disso, apresentar como "não identificado" para o usuário vincular manualmente.

### 3. Backend — OFX Parser (`OfxParserService`)

O formato OFX é um SGML semi-estruturado. **Não usar bibliotecas externas de OFX** — implementar parser próprio simples com `String` manipulation para evitar dependências:

```java
@Service
public class OfxParserService {

    public OfxDadosDto parse(MultipartFile arquivo) throws IOException {
        String conteudo = new String(arquivo.getBytes(), StandardCharsets.UTF_8)
            .replace("\r\n", "\n").replace("\r", "\n");

        OfxDadosDto dados = new OfxDadosDto();

        // Extrair metadados da conta
        dados.setBancoId(extrairValor(conteudo, "BANKID"));
        dados.setAgencia(extrairValor(conteudo, "BRANCHID"));
        dados.setNumeroConta(extrairValor(conteudo, "ACCTID"));

        // Extrair período
        String dtStart = extrairValor(conteudo, "DTSTART");
        String dtEnd = extrairValor(conteudo, "DTEND");
        dados.setDataInicio(parseOfxDate(dtStart));
        dados.setDataFim(parseOfxDate(dtEnd));

        // Extrair transações (STMTTRN blocks)
        List<OfxTransacaoDto> transacoes = new ArrayList<>();
        Pattern pattern = Pattern.compile(
            "<STMTTRN>(.*?)</STMTTRN>",
            Pattern.DOTALL | Pattern.CASE_INSENSITIVE
        );
        Matcher matcher = pattern.matcher(conteudo);
        while (matcher.find()) {
            String bloco = matcher.group(1);
            OfxTransacaoDto t = new OfxTransacaoDto();
            t.setOfxId(extrairValor(bloco, "FITID"));
            t.setTipo(extrairValor(bloco, "TRNTYPE"));
            t.setValor(new BigDecimal(extrairValor(bloco, "TRNAMT").replace(",", ".")));
            t.setData(parseOfxDate(extrairValor(bloco, "DTPOSTED")));
            t.setMemo(extrairValor(bloco, "MEMO"));
            t.setNome(extrairValor(bloco, "NAME"));
            transacoes.add(t);
        }
        dados.setTransacoes(transacoes);
        return dados;
    }

    private String extrairValor(String texto, String tag) {
        Pattern p = Pattern.compile("<" + tag + ">([^\n<]+)", Pattern.CASE_INSENSITIVE);
        Matcher m = p.matcher(texto);
        return m.find() ? m.group(1).trim() : "";
    }

    private LocalDate parseOfxDate(String ofxDate) {
        if (ofxDate == null || ofxDate.length() < 8) return null;
        // OFX date format: YYYYMMDD[HHMMSS[.XXX][Z]]
        String d = ofxDate.substring(0, 8);
        return LocalDate.parse(d, DateTimeFormatter.ofPattern("yyyyMMdd"));
    }
}
```

**DTOs internos do parser:**
- `OfxDadosDto` — metadados + lista de transações
- `OfxTransacaoDto` — ofxId, tipo, valor, data, memo, nome

### 4. Backend — Algoritmo de Score (`ConciliacaoService`)

```java
private int calcularScore(OfxTransacaoDto ofxTrn, Conta conta) {
    int score = 0;
    BigDecimal valorOfx = ofxTrn.getValor().abs();
    BigDecimal valorConta = conta.getValorOriginal().abs();

    // Critério 1: Valor
    if (valorOfx.compareTo(valorConta) == 0) {
        score += 40;
    } else if (valorOfx.subtract(valorConta).abs().compareTo(new BigDecimal("0.05")) <= 0) {
        score += 20;
    }

    // Critério 2: Data (±3 dias)
    if (ofxTrn.getData() != null && conta.getDataVencimento() != null) {
        long diasDiferenca = Math.abs(
            ChronoUnit.DAYS.between(ofxTrn.getData(), conta.getDataVencimento())
        );
        if (diasDiferenca == 0) score += 30;
        else if (diasDiferenca <= 3) score += 15;
    }

    // Critério 3: Similaridade textual MEMO × descricao
    if (ofxTrn.getMemo() != null && conta.getDescricao() != null) {
        double similaridade = calcularJaccard(
            normalizar(ofxTrn.getMemo()),
            normalizar(conta.getDescricao())
        );
        if (similaridade >= 0.6) score += 20;
        else if (similaridade >= 0.3) score += 10;
    }

    // Critério 4: Parceiro
    if (conta.getParceiro() != null && ofxTrn.getMemo() != null) {
        String nomeParc = conta.getParceiro().getNomeFantasia() != null
            ? conta.getParceiro().getNomeFantasia()
            : conta.getParceiro().getRazaoSocial();
        if (nomeParc != null && normalizar(ofxTrn.getMemo())
                .contains(normalizar(nomeParc).substring(0, Math.min(5, nomeParc.length())))) {
            score += 10;
        }
    }

    return score;
}

// Jaccard similarity sobre bigramas
private double calcularJaccard(String a, String b) {
    Set<String> bigramasA = bigramas(a);
    Set<String> bigramasB = bigramas(b);
    if (bigramasA.isEmpty() || bigramasB.isEmpty()) return 0;
    Set<String> intersecao = new HashSet<>(bigramasA);
    intersecao.retainAll(bigramasB);
    Set<String> uniao = new HashSet<>(bigramasA);
    uniao.addAll(bigramasB);
    return (double) intersecao.size() / uniao.size();
}

private Set<String> bigramas(String s) {
    Set<String> bg = new HashSet<>();
    for (int i = 0; i < s.length() - 1; i++) bg.add(s.substring(i, i + 2));
    return bg;
}

private String normalizar(String s) {
    return Normalizer.normalize(s, Normalizer.Form.NFD)
        .replaceAll("[^\\p{ASCII}]", "")
        .toLowerCase()
        .replaceAll("[^a-z0-9]", " ")
        .trim();
}
```

### 5. Backend — Endpoints REST

**Base:** `/api/conciliacao`

| Método | Path | Descrição |
|--------|------|-----------|
| `POST` | `/api/conciliacao/upload` | Upload OFX → cria sessão + retorna análise |
| `GET` | `/api/conciliacao` | Lista histório de conciliações do usuário |
| `GET` | `/api/conciliacao/{id}` | Busca sessão com todos os itens |
| `PUT` | `/api/conciliacao/{id}/itens/{itemId}/vincular` | Vincula manualmente item a uma conta |
| `POST` | `/api/conciliacao/{id}/itens/{itemId}/nova-conta` | Cria conta a pagar/receber e vincula ao item |
| `PUT` | `/api/conciliacao/{id}/itens/{itemId}/ignorar` | Marca item como ignorado |
| `POST` | `/api/conciliacao/{id}/finalizar` | Confirma conciliação → dispara todas as baixas |
| `DELETE`| `/api/conciliacao/{id}` | Cancela sessão pendente |

#### `POST /api/conciliacao/upload` — Lógica do Serviço

1. Faz parse do OFX (`OfxParserService`)
2. Identifica a `ContaCorrente` pelo número de conta do OFX:
   - Se encontrar → usa automaticamente
   - Se não encontrar → retorna `HTTP 422` com `{ codigoConta: "...", mensagem: "Conta não encontrada" }` para o frontend solicitar ao usuário
3. O frontend então chama novamente enviando `contaCorrenteId` forçado via query param ou body
4. Cria registro em `tb_conciliacao` com `status = 'PENDENTE'`
5. Para cada transação do OFX:
   - Busca contas PENDENTE/ATRASADO do usuário com valor próximo (filtro inicial via SQL: `|valor - ofxValor| <= 10`)
   - Calcula score para cada candidata
   - Score ≥ 50: cria `ConciliacaoItem` com `status = 'SUGERIDO'` e vincula a `conta_id` com maior score
   - Score < 50: cria `ConciliacaoItem` com `status = 'NAO_IDENTIFICADO'`
6. Atualiza contadores na `tb_conciliacao`
7. Retorna `ConciliacaoResponseDto` completo

#### `POST /api/conciliacao/{id}/finalizar` — Lógica

1. Valida que a sessão pertence ao usuário e está `PENDENTE`
2. Para cada `ConciliacaoItem` com status `SUGERIDO` ou `VINCULADO_MANUALMENTE`:
   - Chama `ContaService.baixar()` passando `contaCorrenteId` da sessão e `dataPagamento = ofx_data`
   - Atualiza `conta_baixada_id` e `status_item = 'BAIXADO'`
3. Atualiza `tb_conciliacao.status = 'FINALIZADA'` e `data_conciliacao = now()`
4. Retorna resumo final

### 9. Segurança

- Todos os endpoints verificam `usuario_id` via `SecurityContextUtils`
- Upload limitado: validar extensão `.ofx` e tamanho ≤ 5MB no backend (`MultipartFile`)
- Arquivo OFX nunca é persistido em disco — processar em memória
- Ao finalizar, utilizar transação `@Transactional` única para todas as baixas (rollback total em caso de erro)

---

### 10. Tratamento de Erros

| Situação | Resposta |
|---|---|
| Arquivo não é OFX válido | 422 `OFX_INVALIDO` |
| Conta corrente não encontrada no OFX | 422 `CONTA_NAO_ENCONTRADA` com `numeroConta` |
| Sessão já finalizada | 400 `CONCILIACAO_JA_FINALIZADA` |
| Conta já baixada em outra conciliação | 409 `CONTA_JA_BAIXADA` |
| Score < threshold para todas as candidatas | item salvo como `NAO_IDENTIFICADO` (não é erro) |

---

## Observações Finais

1. **Não usar biblioteca OFX externa** — o parser próprio é suficiente e evita dependências desnecessárias no `build.gradle`
2. **Atomicidade:** a finalização deve ser uma única `@Transactional` — qualquer falha reverte todas as baixas
3. **Idempotência:** um mesmo `ofx_id` + `conta_corrente_id` nunca deve gerar dois `ConciliacaoItem` — adicionar constraint única se necessário
4. **UX:** o usuário nunca perde dados ao navegar — salvar estado intermediário na sessão de conciliação (`PENDENTE`) no banco
5. **Relatório:** seguir exatamente o padrão de `RelatorioService.gerarExcel()` existente para exportação
6. **Siga os padrões visuais** do design system existente: `bg-surface-medium`, `rounded-2xl`, `border border-text-primary/5`, `font-display`, `font-body`, badges via `Badge.jsx`, confirmações via `useConfirm`, toasts via `useToast`
